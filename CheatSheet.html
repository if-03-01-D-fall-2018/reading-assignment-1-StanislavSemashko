World Wide Web General!

1988:

The first direct IP connection between Europe and North America was made.

1989:

A proposal for a System called Mesh which described a more elaberate information management based on links in a readable text/hyperlinks

1990:

Hypertextproject called WorldWideWeb as a web of hypertext documents which are viewed by browsers.

1993:

The World Wide Web got free to anyone and the Mosaic Webbrowser was released.

(WorldWideWeb != Internet)

Reasons why the World Wide Web was such a success:

The Internet and Hypertext were married for the first time. Also these three technologies were essential for the development of the World Wide Web. The URL (uniform resource locator) and URI(uniform resource identifier), or former UDI (universal document identifier) is a system for (globally) unique identifiers for resources on the Web.

The general components of a url:

1. A scheme. The scheme identifies the protocol to be used to access the resource on the Internet. It can be HTTP (without SSL) or HTTPS (with SSL).

2. A host. The host name identifies the host that holds the resource. For example, www.example.com.

3. A path. The path identifies the specific resource within the host that the Web client wants to access. For example, /software/htp/cics/index.html.

HTML... Hypertext Markup Language

HTTP...Hypertext Transfer Protocol

How resources (html pages, images, etc) addressed in the web:

The Web is an information space containing hyperlinked documents and other resources, identified by their URIs.It is implemented as both client and server software using Internet protocols such as TCP/IP and HTTP.

Web Server!

What is a web server:

A webserver shall store, process and deliver webpages to clients. The communication between the server and the client takes place using http. Mostly delivered are HTML Documents.

What does a web server:

For serving the World Wide Web. They can also be found embedded in devices such as printers, routers, webcams and serving only a local network.

A web browser or a web crawler starts to communicate by making a request for a specific resource using HTTP or HTTPS. The server responds with the resource or an error.

How can content be stored on a web server:

While the primary function is to serve content, a full implementation of HTTP also includes ways of receiving content from clients. This feature is used for submitting web forms, including uploading of files.

How is content in webpages described:

Many generic web servers also support server-side scripting using Active Server Pages (ASP), PHP, or other scripting languages.

Web Client!

What is a web client:

Also known as Web Browser, a web client is an application for accessing information on the world wide web.

What does a web client:

It fetches information resources and displays them to an end device like a computer.

The process is the following: The user inputs a URL, the rendering engine of the browser renders it and then displays it.

Hello World!

Doctype:

The <!DOCTYPE> on the head of a html document exists to show the browser the version of html that is used. It's not always required but you should still use it always.

Body element:

The <body> element contains all of the contents such as text, images and other.

em element:

The <em> tag is used to show important text that should be higlighted from the rest of the text

Head element:

The <head> element contains all of the metadata and things like the <style> tag, the <script> tag or the <link> tag.

html tag:

The <html> element shows the browser that the following document is a html document. It also is the root for all of the other tags and elements of the html document.

strong element:

The <strong> exists for basically the same reason as the <em> tag. It just has a other style, it's kinda bold, which is also very useful sometimes.

title element:

The <title> element is required in a html document, it is basically just the title of the document. It is between the <head> tags.
